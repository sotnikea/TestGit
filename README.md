> Ігнорування файлів
 
Приклади: https://github.com/github/gitignore 

| Команда  | Опис |
| --------------- | -------- |
| cat .gitignore | Створити gitignore файл. Далі зміни вносяться безпосередньо до створеного файлу |
| *.[oa] | ігнорувати файли, що закінчуються на “.o” або “.a” |
| *~  | ігнорувати всі файли, назви яких закінчуються на тильду |
| *.a | ігнорувати файли, що закінчуються на “.a” |
| !lib.a | відстежувати !lib.a, враховуючи ігнорування всіх .а |
| build/  | Ігнорувати усі файли в теці build/ |
| doc/*.txt | Ігнорувати doc/notes.txt, проте не doc/server/arch.txt |
| doc/**/*.pdf | Ігнорувати усі .pdf файли в теці doc/ та всіх її підтеках |


> Відображення не індексованих змін

| Команда  | Опис |
| --------------- | -------- |
| git diff | Демонструє зроблені зміни, які ще не індексовані |
| git diff --staged | Демонструє індексовані зміни |
| git diff --cached | Демонструє індексовані зміни |

> Створення коміту

| Команда  | Опис |
| --------------- | -------- |
| git commit | Створення коміту через редактор |
| git commit -v   | Через редактор + детальна інфа по змінам |
| git commit -m "Текст коміту" | Створення коміту з консолі |
| git commit -a  | Через редактор + додає всі tracked але змінені файли до коміту |
| git commit -a -m 'Текст коміту' | З консолі + додає всі tracked але змінені файли до коміту |

> Видалення файлу з індексу

| Команда  | Опис |
| --------------- | -------- |
| git rm <Назва файлу> | Видаляє файл та індексує його на видалення |
| git rm -f <Назва файлу>  | Якщо треба видалити редагований доданий до індексу файл |
| git rm --cached <Назва файлу> | Індексує файл на видалення, але сам файл не видаляє |
| git rm log/\*.log  | Видаляє всі файли, що мають .log розширення та знаходяться в директорії log/ |
| git rm \*~  | Видаляє всі файли, чиї назви закінчуються на ~ |

> Перегляд історії комітів

Для форматованого виводу детальний опис: https://git-scm.com/book/uk/v2/%D0%9E%D1%81%D0%BD%D0%BE%D0%B2%D0%B8-Git-%D0%9F%D0%B5%D1%80%D0%B5%D0%B3%D0%BB%D1%8F%D0%B4-%D1%96%D1%81%D1%82%D0%BE%D1%80%D1%96%D1%97-%D0%BA%D0%BE%D0%BC%D1%96%D1%82%D1%96%D0%B2#pretty_format 

| Команда  | Опис |
| --------------- | -------- |
| git log  | Перегляд історії комітів |
| git log -p -2  | -p - патч, різниця внесена в кожному коміти, -2 - 2 останніх коміти |
| git log --stat | Скорочена інформація  |
| git log --pretty=oneline | Форматований ввід (online, short, full, fuller, format) |
| git log --since=2.weeks | Коміти за останні 2 тижні |
| git log --oneline --decorate | Демонструє куди вказують вказівники гілок |
| git log --oneline --decorate --graph --all | Малює простий граф з інфою які гілки які коміти містять |

> Скасування речей

| Команда  | Опис |
| --------------- | -------- |
| git commit --amend  | Корегування останнього коміту |

| git checkout -- <назва файлу> | Відкат файлу до стану останнього коміту |

> Взаємодія з віддаленими сховищами

| Команда  | Опис |
| --------------- | -------- |
| git remote   | Відображає налаштовані віддалені репозиторії |
| git remote -v  | Відобразити посилання на репозиторій |
| git remote add <ім’я> <посилання>  | Додати новий репозиторій |
| git fetch <remote>  | Скачує всі зміни з указаного репозиторію, але не зливає їх |
| git pull   | Зливає зміни з віддаленого репозиторію до локального |
| git push origin main | Надсилання змін до віддаленої гілки |
| git remote show origin  | Показує, до яких гілок автоматично надсилаються зміни git push |
| git remote rename <стара назва> <нова> | Зміна назви для віддаленого репозиторія |

> Тегування

| Команда  | Опис |
| --------------- | -------- |
| git tag | Отримати список доступних теґів у Git |
| git tag -l "текст_для_пошуку" | Пошук тегів за шаблоном |
| git tag -a назва_тегу -m "опис_тегу" | Додати анотований тег. а - add, додавання . m - використати опис для тегу |
| git show назва_тегу | Демонструє інформацію пов'язану з тегом |
| git tag назва_тегу | Створює легкий тег. Містить лише коміт без додаткової інформації |
| git tag -a назва_тегу хеш_коміту | Додає тег до створеного раніше коміту. Без -m доведеться додавати опис в редакторі |
| git push назва_гілки назва_тегу | Відправка вказаного тегу до віддаленого репозиторію |
| git push назва_гілки --tags | Відправка всіх тегів |
| git checkout назва_тегу | Перемикання гілки до вказаного тегу. Однак отримаємо стан detached HEAD. При будь яких змінах теґ залишиться незмінним, а новий коміт не належатиме жодній гілці й буде досяжним лише за допомогою хеша коміту |

> Псевдоніми Git

| Команда  | Опис |
| --------------- | -------- |
| git config --global alias.ci commit | Створити псевдонім ci для команди commit. Можна буде використовувати ci замість commit |
| git config --global alias.last 'log -1 HEAD' | Створити для log -1 HEAD псевдонім last щоб відображати останній коміт командою git last |

## Галуження в git

> Основи галуження та зливання

| Команда  | Опис |
| --------------- | -------- |
| git branch | Перелік існуючих гілок |
| git branch назва_гілки | Створює нову гілку, або іншими словами створює новий вказівник на фіксацію, в якій ми зараз знаходимось |
| git checkout назва_гілки | Переключення на вказану гілку |
| git checkout -b назва_гілки | Створити нову гілку і одразу перемкнутись на неї |
| git merge назва_гілки | Змержити указану гілку в поточну |
| git branch -d назва_гілки | Видалення гілки, яка більше не потрібна |

> Управління гілками

| Команда  | Опис |
| --------------- | -------- |
| git branch -v | Останній коміт на гілці |
| git branch --merged | Перелік гілок змержених з поточною |
| git branch --no-merged | Перелік гілок не змержених з поточною |

> Віддалені гілки

| Команда  | Опис |
| --------------- | -------- |
| git ls-remote | Відображення віддалених гілок та тегів |
| git remote show | Відображення віддалених гілок |
| git clone адреса | Клонування з віддаленого репозиторію в origin/master |
| git clone -o назва_посилання адреса | Клонування з віддаленого репозиторію в назва_посилання/master |
| git fetch origin | Шукає який сервер відповідає імені "origin", отримує дані, яких локально ще немає і оновлює локальну базу даних, переміщуючи вказівник origin/master на нову, більш актуальну, позицію |
| git push назва_посилання назва_гілки | Відправка конкретної гілки на сервер. Назва_посилання за замовчуванням origin, можна не писати якщо не змінювалось на іншу назву |
| git merge origin/назва_гілки | Злити віддалену гілку в локальну поточну |
| git checkout -b назва_гілки origin/назва_гілки | Створити локальну гілку на основі віддаленої гілки |
| git branch -u origin/назва_гілки | Прив'язати поточну локальну гілку до віддаленої, щоб отримувати з неї оновлення |
| git branch -vv | Виводить інформацію, до яких віддалених гілок прив'язані локальні та звідки отримують оновлення |
| git fetch --all | Оновлює всі посилання |
| git pull | Фактично послідовно виконує команди git fetch та git merge в локальній гілці витягаючи дані з віддаленою прив'язаної гілки |
| git push origin --delete назва_гілки | Видалення віддаленої гілки |

> Перебазовування

| Команда  | Опис |
| --------------- | -------- |
| git rebase master | Поточну гілку перебазовуємо на указану (В даному випадку на master). Фактично зміни поточної гілки переносяться в кінець указаної гілки(master). Після цього її можна наприклад змержити |
| git rebase --onto master гілка_1 гілка_1_1 | Результат: Взяти гілку_1_1 ті її латки, які з'явились від моменту відгалудження від гілки_1 та відтворити її наче вона на гілці master. Далі можна мержити з master |
| git rebase master назва_гілки | Перебазувати указану гілку на master. Після цього можна її мержити |

| Команда  | Опис |
| --------------- | -------- |
